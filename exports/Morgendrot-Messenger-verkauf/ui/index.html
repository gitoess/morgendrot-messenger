Transparenz & Schutz

Morgendrot soll intuitiv und sicher im Sinne von E2EE sein – ohne falsche Versprechen zur Anonymität.

  • E2EE gilt für den Nachrichteninhalt (nach Handshake/Connect): Inhalt ist für Dritte nicht lesbar; Metadaten auf der Chain (Adressen, dass etwas gesendet wurde, Mailbox/Events, Gas) sind öffentlich sichtbar wie bei jeder öffentlichen Blockchain.
  • Kein NFT nötig für normalen Chat. Optional später: Komfortpakete – der freie Weg bleibt: Package + RPC + Wallet + Gas.
  • API & UI binden an 127.0.0.1 – sensible Eingaben bleiben lokal; RPC-Betreiber können Anfragen sehen → Stufen 1–3 (Rotation, Proxy, eigener Node) darunter.
  • Tresor empfohlen für Laien (Keys wiederverwenden); ohne Tresor möglich für versierte Nutzer.

Netzwerk-Vertrauen & RPC-Rotation

Stufe 1 = öffentliche RPCs (Rotation wechselt den Anbieter, nicht das Blockchain-Risiko). Stufe 2 = RPC_HTTP_PROXY in .env (z. B. HTTP-Proxy vor Tor). Stufe 3 = eigener Node. Anzeige aus NETWORK_TRUST_TIER (Hinweis für dich, keine technische Durchsetzung).
